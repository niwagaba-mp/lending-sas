1. Product Overview

SMOS (Smart Microfinance Operating System) is a cloud-native, multi-tenant SaaS platform purpose-built for decentralized microfinance institutions operating across multiple branches. It automates the full lending lifecycle — from client registration and loan origination through to repayment collection, arrears classification, credit scoring, legal recovery, and executive reporting.

The platform is designed for the East African microfinance context, with built-in GPS verification, anti-ghost client mechanisms, staff traceability, and Ugandan Shilling (UGX) currency support.

Key Value Propositions

Multi-Tenant SaaSA single deployment serves multiple independent microfinance institutions, each fully isolated with their own branches, staff, clients, and data
Decentralized OperationsSupports geographically distributed branches, each with their own manager, loan officers, and cashiers

4. Core Modules

4.1 Authentication & Security

API Base: /api/auth

  • Login: Email + password authentication with JWT issuance
  • Refresh Token Rotation: Old refresh token is revoked each time a new access token is issued, preventing replay attacks
  • Password Management: POST /auth/change-password with current password verification and minimum length enforcement (8+ characters)
  • Logout: Explicit refresh token invalidation
  • Rate Limiting: Auth endpoints limited to 20 requests per 15 minutes
  • Session Info: GET /auth/me returns current user profile

4.2 Branch Management

API Base: /api/branches

Manages the geographical structure of the microfinance institution.

  • Branch Profiles: Name, code, address, district, region, GPS coordinates, phone, email
  • Manager Assignment: Each branch has a designated branch manager
  • Operational Metrics (computed on-the-fly): client count, staff count, active loan count, portfolio value
  • Tenant Isolation: All queries filtered by tenant_id

4.3 Staff Management

API Base: /api/staff

Comprehensive HR and identity management for microfinance field staff.

  • KYC Data: National ID, gender, date of birth, dual addresses (village + urban), GPS coordinates
  • Traceability: Recommender details, internal referees, guarantors with GPS
  • Local Authority Contact: LC1 chairman name, phone, and address for verification
  • Incident Tracking: Fraud case flags, absconding flags, audit failure counts, discrepancy counts
  • Secure Onboarding: Random password generation on staff creation (returned to admin)
  • Staff P&L View: Per-officer profit/loss — portfolio size, interest earned, defaults, expenses, net profit
  • Staff-Client Registry: All clients assigned to a specific loan officer
  • Leaderboard: Performance ranking across the branch

4.4 Client Management

API Base: /api/clients

  • Full KYC Profile: Name, national ID, gender, DOB, education, marital status, dependants, business info
  • GPS Verification: Mandatory home GPS coordinates; optional business GPS
  • Anti-Ghost Flags: gps_captured and photo_verified booleans track field verification
  • Supervisor Approval Flow: Clients must be approved by a supervisor before receiving loans
  • Document Storage: Passport photo, ID front/back, and other documents (URL references)
  • Next of Kin: Multiple next-of-kin records per client
  • Client Map View: Geographic visualization of all registered clients
  • Client CRM: Cashier-facing module for client relationship management

4.5 Loan Lifecycle

API Base: /api/loans

The core lending engine with a 12-state lifecycle:

Draft → Pending Approval → Approved → Disbursed / Active → At Risk (1-7d) → Delinquent (8-30d) → Defaulted (31-90d) → Non-Performing (90+d)
Active → Closed (Fully Repaid)    |    Non-Performing → Written Off

Loan Terms

  • Principal amount with flat interest rate
  • Processing fee
  • Repayment frequency: daily, weekly, or monthly
  • Duration in days with computed installment count and amount
  • Auto-generated repayment schedule on disbursement

4.6 Repayment Engine

API Base: /api/repayments

  • Payment Recording: Amount, date, collection GPS, payment method (cash, mobile money, bank), reference number
  • Schedule Waterfall: Each payment is applied to overdue installments first, then to the next due installment
  • Loan Balance Sync: After every payment, the loan's total_paid, outstanding_balance, arrears_amount, and arrears_days are recalculated
  • Auto-Status Update: If fully repaid, the loan status automatically transitions to closed
  • Schedule View: Full installment schedule with overdue flags and days-overdue calculation

4.7 Automated Arrears Classification

A node-cron job runs every day at 02:00 AM and processes all active loans:

Arrears DaysStatus Classification
0active
1 – 7at_risk
8 – 30delinquent
31 – 90defaulted
90+non_performing
Balance ≤ 0closed

The engine calculates arrears by comparing expected payments-to-date against actual payments received, then determines days since the first missed installment.

4.8 Credit Scoring Engine

API Base: /api/credit

A six-factor scoring algorithm producing a 0–1000 score:

4.9 Expense Tracking

API Base: /api/expenses

  • Categories: Transport, Airtime, Recovery, Operational, Other
  • Approval Workflow: Expenses must be approved by a manager
  • Linkage: Expenses can be linked to specific loans or clients for P&L attribution
  • Reversal: Managers can reverse expenses with reason tracking
  • Staff Summary: Aggregated view by category per staff member

4.10 Transaction Management

API Base: /api/transactions, /api/misc

Unified Transaction Ledger

A database view (v_transactions) merges four transaction sources into a single timeline:

  • Repayments (cash inflows)
  • Expenses (cash outflows)
  • Disbursements (cash outflows)
  • Miscellaneous — shortages, excesses, fines, unknown funds, loan returns

Transaction Reversal & Editing

  • Managers can reverse repayments or expenses with mandatory reason
  • Transaction type is strictly whitelisted to prevent injection
  • Full audit trail (old → new values) logged on every edit

4.11 Legal Recovery

API Base: /api/legal, /api/legal-recovery

  • Case Filing: Automatic case number generation, linked to loan/client/staff/branch
  • Status Workflow: open → in_progress → resolved / escalated → closed
  • Court Integration: Court name, reference number, hearing dates
  • Financial Tracking: Outstanding balance and arrears at time of filing
  • Report Generator: Professional legal documents for court proceedings

4.12 Notification System

API Base: /api/notifications

  • Channels: SMS, WhatsApp, Email, System notifications
  • Automated Reminders: Batch overdue payment reminders with one-click send
  • Notification Log: Full history with delivery status (pending, sent, failed)

4.14 SaaS Control Plane

API Base: /api/superadmin — Super Admin only

  • Tenant Management: Create new microfinance institutions with initial admin, branch, and database setup (transactional)
  • Subscription Billing: Plan management, billing amounts, billing cycles, grace periods
  • Tenant Lock/Unlock: Manually suspend or reactivate tenant accounts
  • System Revenue Tracking: Subscription payments from all tenants
  • System Owner Expenses: Track platform overhead (servers, APIs, staff, legal)
  • Financial Statement View: Revenue vs. expense analysis for the SaaS business
  • Geo-Login Monitoring: Track login activity — IPs, devices, geolocation, success/failure

4.15 Executive Business Console (Business Owner)

A dedicated executive portal for the Tenant Admin (Business Owner) to monitor and compare performance across all branches. Unlike the cashier workstation (which handles daily POS operations) or the supervisor dashboard (which manages operational approvals within a single branch), the Executive Console provides strategic, branch-level comparative analytics for institutional decision-making.

Key Distinction All executive reports are branch-segmented — the owner selects a specific branch and views its isolated performance metrics, or compares multiple branches side-by-side. This is fundamentally different from the cashier's daily transaction view or the supervisor's operational approval queue.

Branch-Level Executive Reports

ReportScopeDescription
Branch DashboardPer branchPortfolio value, active loans, client count, staff performance KPIs, and daily activity — all scoped to a single selected branch
Branch Income StatementPer branchGross interest income, processing fees, itemized expenses, net profit, money in circulation, and cash-at-hand — all for a specific branch over a selected date range
Branch Collection EfficiencyPer branchExpected vs. actual collections per branch, revealing which branches are underperforming
Branch PAR AnalysisPer branchPortfolio-at-Risk (PAR30/60/90) broken down by branch for comparative risk assessment
Staff Arrears RankingPer branchLoan officers ranked by arrears rate within each branch — identifies weakest performers per location
Branch Loan AgingPer branchLoan portfolio health segmented by age, status, and arrears days for each branch

Executive Modules

  • Portfolio Lifecycle Hub: End-to-end loan portfolio tracking with branch selector — view disbursements, active loans, defaults, and closures per branch
  • Staff Performance Appraisals: Data-driven evaluations per branch — collection rates, default rates, client growth, and P&L per loan officer within the selected branch
  • Financial Audit Console: Branch-level financial reconciliation — compare revenue, expenses, and net profit across branches to identify anomalies
  • Periodic Reporting Hub: Configurable time-period reports (daily/weekly/monthly/quarterly) per branch for board presentations and investor updates
Not to be confused with Cashier Workstation — a POS interface for processing daily transactions (repayments, expenses, reconciliation). It has no branch comparison or strategic analytics.
Supervisor Dashboard — an operational approval queue for client sign-offs, staff oversight, and arrears management within a single branch. It does not provide cross-branch financial analysis.

5. Cashier Workstation

A standalone, full-screen point-of-sale interface designed for cashier staff who process payments all day:

  • Repayment Collection: Quick-entry payment processing with loan lookup
  • Daily Cash Reconciliation: Real-time balance of inflows vs. outflows
  • Client CRM: Search clients, view loan histories, audit payment trails
  • Expense Recording: Log operational expenses during the day
  • Transaction Summary: Live financial summary filtered by staff and date
Design Note The cashier workstation operates independently from the admin portal layout — no sidebar, no navigation chrome — just the operational interface optimized for speed and focus.
